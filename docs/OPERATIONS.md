# 运维说明

## 日常检查

建议定期执行：

```bash
npm run build:dataset
npm run predeploy
```

页面侧重点检查：

- 仪表盘数据是否正常；
- 数据源健康检测是否存在大量失败；
- 静态产物是否可访问；
- 配置保存是否需要鉴权；
- 测速导入、筛选、导出是否正常。

## 数据源维护

当源失效或结果异常时：

1. 打开仪表盘执行健康检测；
2. 查看失败源和慢源；
3. 进入配置中心调整 URL 或提取规则；
4. 保存前查看变更摘要；
5. 保存后重新构建数据产物。

## 配置保存

配置中心支持保存前预览和快照归档。推荐流程：

```text
修改配置 → 查看保存预览 → 导出快照 → 确认保存 → 重新检测数据源
```

## 快照管理

本地快照保存在浏览器 localStorage，仅用于当前浏览器的操作留档。

支持：

- 查看快照；
- 搜索快照；
- 按变更类型筛选；
- 导出单条快照；
- 恢复保存前配置；
- 恢复待保存配置；
- 恢复前差异预览；
- 删除或清空快照。

## 测速运维

本地预览可使用 TCP connect 测速。

线上推荐流程：

```text
导出候选 IP → 使用 CloudflareSpeedTest/cfst 测速 → 导入 CSV → 设置质量筛选 → 保存历史 → 导出最终结果
```

测速历史最多保留 30 条，localStorage key：

```text
cf_bestip_speedtest_history_v1
```

## 缓存管理

健康检测和导出接口可能使用缓存。异常时可：

- 页面点击清理缓存；
- 调用 `/api/cache-clear`；
- 使用 `refresh=1` 强制刷新。

清理缓存需要 `ADMIN_TOKEN`。

## 安全要求

- 不要在仓库中保存真实 Token、Cookie 或私有订阅地址。
- `ADMIN_TOKEN` 应使用足够长的随机字符串。
- 线上必须确认管理接口未裸奔。
- 修改 URL 校验逻辑后必须重新运行测试。
